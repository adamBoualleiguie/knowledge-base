Complete guide to adding documentation to your portfolio website with detailed examples.

## Understanding Documentation Structure

Documentation uses a hierarchical structure:

- **Sections** (top level) - Created from folder names
- **Subsections** (second level) - Created from nested folders  
- **Documents** (individual pages) - MDX files

## Step 2: Create Documents with Frontmatter

Every documentation file needs frontmatter:

```mdx
---
title: Your Document Title
description: A brief description of what this document covers
publishedAt: 2024-01-15
updatedAt: 2024-01-20  # Optional
order: 1  # Optional: Controls display order (lower numbers appear first)
---
```

### Required Fields

- **`title`**: Document title (appears in sidebar and page header)
- **`description`**: Brief description (appears in listings and meta tags)
- **`publishedAt`**: Publication date (YYYY-MM-DD format)

### Optional Fields

- **`updatedAt`**: Last update date (YYYY-MM-DD format)
- **`order`**: Display order number (lower numbers appear first, defaults to alphabetical)

## Step 3: Write Your Content

Write content using Markdown or MDX:

```mdx
---
title: Installation Guide
description: Step-by-step installation instructions
publishedAt: 2024-01-15
---

# Installation Guide

This is the main heading (H1). It appears as the page title.

## Introduction

This is a section heading (H2). It will appear in the table of contents.

### Prerequisites

Before you begin, make sure you have:

- Node.js 20.19.4 or higher
- Yarn package manager
- Git installed

## Installation Steps

### Step 1: Clone the Repository

```bash
git clone https://github.com/yourusername/yourproject.git
cd yourproject
```

### Step 2: Install Dependencies

```bash
yarn install
```

### Step 3: Start Development Server

```bash
yarn dev
```

## Verification

To verify the installation worked:

1. Open your browser
2. Navigate to `http://localhost:3000`
3. You should see the homepage

## Next Steps

- Read the [Configuration Guide](/docs/guide/customization/overview)
- Check out [Features](/docs/guide/features/overview)
```

## Step 4: Set Document Order

Control the display order of your documents using the `order` field:

```mdx
---
title: Getting Started
description: Quick start guide
publishedAt: 2024-01-01
order: 1  # This document appears first
---
```

**Ordering Rules:**
- Lower numbers appear first (1, 2, 3...)
- Documents without `order` are sorted alphabetically at the end
- Use `order: 1` for overview pages
- Leave gaps (10, 20, 30...) if you might add documents in between

**Example:**
```mdx
---
title: Overview
order: 1
---

---
title: Installation
order: 2
---

---
title: Configuration
order: 3
---
```

For complete ordering setup (sections, subsections, and documents), see [Documentation Ordering System](/docs/guide/customization/section-ordering).

## Step 5: Overview Pages

Create overview pages for major sections to make section titles clickable:

```mdx
---
title: API Documentation
description: Complete API reference and endpoint documentation
publishedAt: 2024-01-01
---

# API Documentation

Welcome to the API documentation section.

## What's in this section

- **Endpoints**: Detailed documentation for all API endpoints
- **Reference**: Complete API reference
- **Authentication**: How to authenticate

## Quick Links

- [API Endpoints](/docs/api/endpoints)
- [API Reference](/docs/api/reference)
```

**File location**: `content/docs/api/overview.mdx`

**Result**: Section title "API" becomes clickable in sidebar

## Best Practices

1. **Use descriptive titles**: Make titles clear and specific
2. **Add descriptions**: Help users understand what the document covers
3. **Use headings**: Structure content with H2, H3, H4 headings
4. **Link between docs**: Use internal links to connect related content
5. **Create overview pages**: Add overview pages for major sections
6. **Use examples**: Include code examples and screenshots
7. **Keep it organized**: Group related documents in the same section
